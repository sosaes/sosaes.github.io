Mobile Apps:



Kung Fu Runner

Release Date: August 2017

Technologies: Unity, C#, Blender, Photoshop, Audacity, Lightworks

Summary:
This was my second published project... and it was a tough one. After publishing a very simple game, Ms. PacBird, I wanted to create something more complex. I had never worked with 3D modelling or animation before, so I had to learn how to use Blender from scratch. On average, it took me around 15 hours to create each character, so it was a pretty lengthy process. I was proud in the end though, as I had finally created something that not only looked pretty cool, but could also move! For this game, I wanted to use the Unity game engine. That involved a few more hours of online courses and tutorials in order to know what was actually going on. Oh, I had also never worked with C# before, so that was another thing to learn haha. Later on, I realized that I would also need 2D assets for the icons, buttons, backgrounds, etc. Sooo it was time to learn how to use Photoshop... which meant more tutorials! And don't get me wrong, I am not complaining. Learning all these new technologies was very cool and rewarding. I can't say that I loved every minute of it, as many processes were kind of tedious (like adding the black border to every single frame of every single character animation), but in the end I loved seeing all my hard work pay off. It took me aroung 3 months to complete this game. It may seem like a simple 2D infinite runner, but the hours I had to put in in order to learn all this on my own, added to the actual time on design and development, prove that it is way harder than it looks! After finishing this app, I was excited to tackle new challenges, which brought me to my next projects...

Ms. PacBird

Release date:

February 2017

Technologies:

Android Studio, libGDX, Java, MS Paint

Summary:
I took my first Computer Science class in college. Before this, I had never really looked into programming. As soon as I took this class, I became very passionate about the topic. When the course finished, not only did I declare my major to be CS (I came to college undeclared), but I also wanted to learn new things on my own. I had always liked videogames, and the idea of creating a mobile app really caught my attention. I did some research online and I came across some Android Studio tutorials. After a few weeks of learning, I downloaded Android Studio and decided to start working. I also found out about the libGDX library, which helps in the development of games. Back then, I had no knowledge of Photoshop or any animation software, so I went with MS Paint. I draw all my elements in Paint and developed the game in Java (my introductory class was in C++, so I had to learn Java as well). After around one month, I published my first game to the Google Play Store. Given how simple it is, it probably should have taken me a lot less time, but I was just starting, so I was a little green haha.

Untitled

Release Date: TBD

Technologies:

Unity, C#, Intuos Tablet, Photoshop, Multiplayer

Summary:
After Kung Fu Runner, I wanted to keep introducing more complexity to my apps. This game will give more freedom to the player. It is going to be a platformer/shooter. The character will be able to move in all directions and shoot enemies, instead of the simpler 2D runner type. I also love drawing, so I wanted to use this as well. Every single element in the game has been hand drawn using an Intuos drawing tablet in Photoshop. The gameplay is pretty much done, but I wanted to make this game even better by implementing online multiplayer. I also started working on another project (more details in the next slide) and all this, along with school work, is a heavy load. I am planning on releasing it before the end of the year, however.

Untitled Virtual Reality Game

I got my first Virtual Reality set at the end of the Summer (2017) and I immediately got hooked. It all felt so real! After playing a few games (and jumping a few times with a horror game), I knew I wanted to develop my own game. I recently contacted students in the 3D modelling and animation department at my university to help me with the design of the game. After creating a few apps on my own, I realized that I would be able to create this game faster with the help of more experienced animators and modellers. This project is in very early stages, but I am planning on releasing it before the start of the Summer of 2018...
